# 开发任务清单

## 第一阶段：资产盘点

目标：把架构 MD 和两个 ZIP 全部纳入项目资产台账。

任务：

- 读取三个源文件元信息。
- 展开两个 ZIP 的文件清单。
- 统计文件格式、数量、大小。
- 生成 `assets_manifest/raw_assets.csv`。
- 生成 `assets_manifest/asset_summary.md`。
- 标记每个文件的目标模块。

验收：

- 所有文件都有台账记录。
- WPS、ET、RAR、DOC 等特殊格式被标记为待转换，而不是被丢弃。

## 第二阶段：规则库代码化 MVP

目标：先完成 3 类规则的代码化闭环。

优先规则：

1. 性别限制。
2. 年龄限制。
3. 重复收费。

任务：

- 读取国家两库 Excel 表头和样例数据。
- 设计统一规则 schema。
- 实现基础规则执行器接口。
- 实现 3 个规则执行器。
- 绑定 SQL 模板。
- 准备最小金标准测试。

验收：

- 给定结构化条件后，可以生成参数化 SQL。
- 执行器输出包含规则、口径、SQL、解释模板。

## 第三阶段：医保 RAG MVP

目标：先完成一批本省政策文件的可检索知识库。

任务：

- 解析 PDF、DOCX、XLSX、XLS。
- 转换或登记 WPS、ET、DOC、RAR。
- 提取标题、文号、日期、来源文件。
- 切片。
- 建立检索接口。
- 返回政策依据和原文片段。

验收：

- 输入专家问题后，能返回相关政策依据。
- 每条返回都能追溯到原始文件。

## 第四阶段：查询闭环

目标：将规则代码和 RAG 结果接入 H5 查询流程。

任务：

- 专家问题解析。
- RAG 政策依据检索。
- 规则候选匹配。
- 结构化条件生成。
- 专家确认。
- SQL 模板生成。
- 查询任务执行。
- 汇总、明细、图表、导出。

验收：

- 至少 5 个专家问题跑通完整流程。
- 查询结果、规则版本、政策依据、SQL 全部留痕。
